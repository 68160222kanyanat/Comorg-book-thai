# สรุป Chapter 4: Cache Memory (หน่วยความจำแคช)

---

## 1. ภาพรวมระบบหน่วยความจำคอมพิวเตอร์
คอมพิวเตอร์มีการจัดลำดับหน่วยความจำหลายระดับเพื่อความสมดุลระหว่าง ความจุ ความเร็ว และราคา โดยเรียงลำดับจากเร็วไปช้าดังนี้:

1.  **Registers**
2.  **Cache**
3.  **Main Memory (RAM)**
4.  **Disk / SSD**
5.  **Optical / Tape**

> **คุณสมบัติสำคัญ:** เมื่อลำดับชั้นต่ำลง ความจุจะเพิ่มขึ้น แต่ความเร็วจะลดลงและราคาต่อบิตจะถูกลง

---

## 2. Cache Memory คืออะไร
แคชคือหน่วยความจำขนาดเล็กที่มีความเร็วสูงมาก ทำหน้าที่เก็บข้อมูลจาก RAM ที่ CPU เรียกใช้งานบ่อย

* **กระบวนการทำงาน:** เมื่อ CPU ต้องการข้อมูล จะตรวจสอบใน Cache ก่อนเสมอ
* **Cache Hit:** พบข้อมูลใน Cache และส่งให้ CPU ได้ทันที
* **Cache Miss:** ไม่พบข้อมูลใน Cache ระบบต้องดึงจาก RAM มาเก็บไว้ใน Cache แทน
* **หลักการ Locality of Reference:** โปรแกรมมักจะเรียกใช้ข้อมูลเดิมซ้ำๆ หรือข้อมูลที่อยู่ใกล้เคียงกัน เช่น การทำงานใน Loop หรือ Array

---

## 3. โครงสร้างของ Cache
Cache ถูกแบ่งออกเป็นส่วนที่เรียกว่า **Cache Line** ซึ่งภายใน 1 Line ประกอบด้วย:
* **Data:** ข้อมูลที่คัดลอกมา
* **Tag:** ตัวระบุตำแหน่งข้อมูลเพื่อใช้ตรวจสอบ
* **Control bits:** บิตควบคุมสถานะต่างๆ

*หมายเหตุ: ข้อมูลจะถูกย้ายจาก Main Memory มายัง Cache ในรูปแบบของ **Block***

---

## 4. วิธีการ Mapping (RAM ↔ Cache)

| วิธีการ Mapping | ลักษณะการทำงาน | ข้อดี | ข้อเสีย |
| :--- | :--- | :--- | :--- |
| **Direct Mapping** | Block ของ RAM ไปได้แค่ตำแหน่งเดียวที่กำหนดไว้ใน Cache | เร็ว, ฮาร์ดแวร์ไม่ซับซ้อน, ราคาถูก | เกิด Cache Thrashing (การแย่งที่กัน) ได้ง่าย |
| **Fully Associative** | Block สามารถไปวางที่ Cache line ไหนก็ได้ที่ว่างอยู่ | มีอัตราการ Hit สูงที่สุด | ฮาร์ดแวร์มีความซับซ้อนและราคาสูง |
| **Set Associative** | แบ่ง Cache เป็น Set และในแต่ละ Set จะมีหลาย Line | ลดปัญหา Thrashing และเป็นวิธีที่นิยมใช้จริง | - |

---

## 5. Replacement Algorithm
เมื่อ Cache เต็มและต้องการนำข้อมูลใหม่เข้ามา ต้องเลือกข้อมูลเดิมออก:
* **LRU (Least Recently Used):** ลบข้อมูลที่ไม่ถูกใช้งานนานที่สุด
* **FIFO (First-In First-Out):** ลบข้อมูลที่เข้ามาถึงก่อน
* **LFU (Least Frequently Used):** ลบข้อมูลที่ถูกใช้งานน้อยรอบที่สุด
* **Random:** สุ่มเลือกข้อมูลที่จะลบออก

---

## 6. Write Policy (นโยบายการเขียนข้อมูล)
* **Write Through:** เขียนข้อมูลลงทั้งใน Cache และ RAM พร้อมกัน (ข้อมูลใน RAM จะถูกต้องเสมอ แต่ทำงานช้ากว่า)
* **Write Back:** เขียนลง Cache ก่อน แล้วค่อยเขียนกลับลง RAM เมื่อมีการเปลี่ยน Block (ทำงานเร็วกว่า แต่ข้อมูลใน RAM อาจไม่อัปเดตทันที)

---

## 7. Multi-Level Cache
การทำงานจะเรียงลำดับจาก: **CPU → L1 → L2 → L3 → RAM**

* **L1 Cache:** อยู่ในตัว CPU, ความเร็วสูงสุด แต่ขนาดเล็กที่สุด
* **L2 Cache:** มีขนาดใหญ่กว่า L1 แต่ความเร็วช้ากว่า
* **L3 Cache:** ขนาดใหญ่ที่สุด และมักถูกใช้ร่วมกันระหว่างหลาย CPU Core

---

**สรุปสั้น:** Cache Memory คือหน่วยความจำความเร็วสูงที่ช่วยลดเวลาการเข้าถึง RAM โดยการเก็บข้อมูลที่ CPU ใช้งานบ่อยไว้ใกล้ตัวที่สุด
