# 4.4 การจัดองค์ประกอบแคชของ Pentium 4 (Pentium 4 Cache Organization)

Pentium 4 ใช้สถาปัตยกรรม **NetBurst** ซึ่งมีการออกแบบหน่วยความจำแคชที่แตกต่างจาก CPU ยุคก่อนหน้าอย่างสิ้นเชิง เพื่อรองรับ Pipeline ที่ยาวมากและความเร็วสัญญาณนาฬิกาที่สูง

---

## 1. L1 Execution Trace Cache
แทนที่จะเก็บคำสั่งดิบ (Instruction) เหมือน CPU ทั่วไป Pentium 4 ใช้ระบบ **Trace Cache**:
* **ลักษณะการเก็บ:** เก็บคำสั่งที่ผ่านการถอดรหัส (Decode) แล้วในรูปแบบ **Micro-ops**.
* **ความจุ:** ประมาณ 12,000 Micro-ops.
* **ข้อดี:** * ไม่ต้องเสียเวลา Decode คำสั่งซ้ำเมื่อเกิด Loop.
    * ลดปัญหาความหน่วงใน Pipeline ที่มีความยาวถึง 20-31 ขั้นตอน.



---

## 2. L1 Data Cache
* **ขนาด:** 8 KB หรือ 16 KB (ขึ้นอยู่กับรุ่น เช่น Northwood หรือ Prescott).
* **เทคโนโลยี:** * **Write-through:** เขียนข้อมูลลง L2 ทันทีเพื่อให้ข้อมูลสอดคล้องกัน.
    * **Low Latency:** ออกแบบให้เข้าถึงได้เร็วระดับ 2-4 Clock Cycles เพื่อให้ทันกับการทำงานของ ALU.

---

## 3. L2 และ L3 Cache (Advanced Transfer Cache)
แคชระดับสูงมีหน้าที่ลด **Miss Rate** หรือการที่ CPU ต้องออกไปดึงข้อมูลจาก RAM ซึ่งช้ากว่ามาก:

| ระดับแคช | รายละเอียด |
| :--- | :--- |
| **L2 Cache** | ขนาด 256 KB - 2 MB เชื่อมต่อด้วยบัส 256-bit รับส่งข้อมูลได้เร็วมาก |
| **L3 Cache** | มีในรุ่น *Extreme Edition* หรือ *Xeon* เพื่อรองรับ Workload หนักๆ |

---

> **Note:** การที่ L1 Data Cache มีขนาดเล็ก เพราะ Pentium 4 เน้นความเร็ว (Speed) มากกว่าปริมาณ (Capacity) เพื่อให้สอดคล้องกับแนวคิด "High Clock Speed" ในยุคนั้น
